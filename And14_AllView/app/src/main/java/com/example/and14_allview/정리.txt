-> 사용하는 이유 : 위젯을 매번 같은 모양으로 xml(Layout)파일에 직접 반복하는것은 매우 오래걸리고
이벤트 처리하는것도 불편함.
(카카오톡 친구 목록을 표현해야함.=> LinearLayout 복붙하는데는 한계가있음)
하나의 모양을 잘 만들어놓고 해당하는 모양으로 패턴을 반복하는 방식의 뷰

1.ListView (목록형)
-세로 방향으로만 목록을 보여줄수가있음.(==대부분의 정보는 사람의 가독성에 맞춰서 세로로 되어있음)
-만드는 순서
    -1. 칸마다 보여질 레이아웃 하나를 만든다. ( 높이값을 부모크기만큼 늘려놓으면 한칸이 너무큼)
    -2. 어댑터를 만든다. ( 일반 클래스를 만들고 나서 => extends BaseAdapter )
        -2.1.LayoutInflater를 반드시 생성자나 메소드를 이용해서 받아와야함(상위 화면에 보이는 클래스로부터)
    -3. 리스트가 있는 화면에 ListView 위젯을 추가하고 해당하는 리스트뷰에 어댑터 연결.
    --------------------내가 원하는 갯수만큼 반복되는 모양을 볼수있음------------------
    -4. 레이아웃 한칸에 있는 위젯중에 내용이 바뀌어야할것을 찾고 해당하는 내용에 대한
    DTO를 만든다 ( DATA )
    -5. 어댑터의 getCount부분이 우리가 임의로 지정한 수가 아니라 List에 size로 바뀜.
    -6. getView 메소드에서 List에 내용과 위젯을 정확히 일치시킨다.

2.GridView (격자형)
-세로 방향만 가능하나 격자형으로 보여줄수있음.(numColums)라는 속성을 이용하면 된다.
그 외에는 모든 방법이 ListView랑 동일함.( 사용빈도가 많이 낮음 )== 디자인작업이 불편함.


3. RecyclerView ( 재활용이 가능한 뷰 ), 가로세로가 전부 다 가능함.
-ListView의 getView부분은 계속해서 실행이 됨.=> 위젯을 찾고 위젯에 데이터를 연결, 이벤트걸기.
->RecyclerView는 계속해서 반복하는게 아니라 처음에 한번 세팅을 하고 재작업을 덜 한다.
